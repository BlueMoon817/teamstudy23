1. 동기와 비동기

  • 웹 브라우저의 특수성

    브라우저는 바로 실행할 수 없는 딜레이가 발생하는 함수들(Web API)을 코드 순서와는 상관 없이 따로 실행한다.

    console.log(1); setTimeout(()=>{ console.log(2)},1000); console.log(3); // 실행결과 1 3 2 // setTimeout은 비동기함수이기 떄문에 코드 작성순서와 상관없이 실행 조건이 맞으면 실행된다. // element.addEventListener('click', function(){}); // $.ajax();

  • JavaScript를 순차적으로 실행하려면 콜백함수를 사용한다.

    console.log(1); setTimeout(function()=>{ console.log(2)},1000); document.addEventListener('click', 함수식별자); console.log(3);

  • 함수는 실행문의 종류에 따라 함수 호출 순서와 상관 없는 순서로 실행될 수 있다.

    함수의 호출순서를 정하고 싶을 때 콜백함수의 형태로 작업한다.

    콜백함수는 함수 디자인 패턴일 뿐 비동기와는 상관 없다.

    function 첫째함수(콜백함수){ console.log('첫번쨰 실행'); 콜백함수(); } function 둘째함수(){ console.log('두번째 실행'); } 첫째함수(둘째함수);

  • 콜백함수의 문제점

    첫째함수(function(){ 둘째함수(function(){ 셋째함수(function(){ }) }) })

  • 콜백함수를 promise 패턴으로 디자인

    첫째함수().then(function(){ 첫쨰함수 실행후 실행할 함수 }).then(function(){ 그다음 실행할 함수 })
