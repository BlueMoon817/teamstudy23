2. Rest Parameter

  • Rest 파라미터 활용

    임시 파라메터를 추가하고 싶은데 arguments배열로는 안묶였으면 하는 경우 rest파라메터를 사용할 수 있다.

    들어올 인자의 개수와 랜덤할 때

    들어온 인자들을 모두 배열로 묶어줌.

    function 함수2(...파라메터들){
      console.log(파라메터들); [1,2,3,4,5,6,7]
    }
    함수2(1,2,3,4,5,6,7);

  • arguments와 차이점

    배열로 묶지 않을 파라미터를 명시할 수 있음(유연함)

    function 함수2(a,b,...파라메터들){
      for(let i=0; i< 파라메터들.length; i+=1){
        console.log(...파라메터들[i]); // 3,4,5,6,7,8
      }
    }
    더하기2(1,2,3,4,5,6,7,8);

  • rest 파라미터 사용의 유의사항

    rest파라메터는 마지막에 사용가능 -> 그 뒤에 다른 파라메터가 올 수 없다.

    1번만 사용할 수 있다.

  • 파라미터

    파라메터는 새로운 변수와 같다.

           따라서 파라메터의 인자를 변수를 넣고 값의 재할당을 시도하는 것은

           파라메터변수 = 인자변수

           즉, 변수1=변수2 와 같은 결과가 나타난다.

           원시값의 경우 파라메터변수에 새 메모리에 동일한 값을 복사되고

           참조값일 경우 파라메터변수에 값이 저장된 메모리 주소를 복사된다.
